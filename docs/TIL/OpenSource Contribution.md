정말 도움이 많이 될 것 같다.1. **빠르게 서비스 개발에 참여해 볼 수 있다.**   이미 만들어지고 있는 서비스에 빠르게 참여해서 전세계의 뛰어나신 다른 개발자 분들의 코드를 보며 참고 및 공부할 수 있고, 테스트 코드를 짜서 직접 테스트도 해보며 공부를 더 효율적으로 할 수 있다.2. **오픈 소스 내부 구조를 더 쉽고 빠르게 파악할 수 있다.**   원래라면, 해당 오픈 소스를 토이 프로젝트에 적용해보고, 어디서부터 어떻게 파악해야 할지 몰라서 docs 파일을 보며 하나하나 테스트를 해가며 어렵게 파악한다. 근데 문제가 되는 이슈를 해결하고자 한다면 해당 이슈와 관련된 코드들을 살펴보게 되고 자연스럽게 전체 프로젝트 구조를 파악할 수 있게 된다는 것을 느꼈다.3. **다른 개발자들과 소통할 수 있다.**   오픈 소스에 기여하다보면 기본적으로 메인테이너와 소통하며 어떤 부분이 문제인지 내가 짠 코드가 잘 짠 코드인지 확인할 수 있고, 더불어서 다른 개발자들의 피드백도 살펴볼 수 있다. 또한, 다른 PR 에 짜여진 코드들을 보면서 다른 개발자들은 어떻게 생각하는지, 내가 미처 생각하지 못하는 부분들을 쉽게 파악하고 얘기를 나눌 수 있다.## 비고- [오픈 소스 톡방](https://open.kakao.com/o/ghrD0mUf)- [[오픈소스 첫 기여 가이드] 오픈소스 멘토링에서 10명 넘는 오픈소스 컨트리뷰터가 첫 기여를 성공할 수 있었던
